# scan-context

上下文健康度诊断。只读模块，不修改任何文件。当 Claude 行为异常时使用此工具定位问题根源。

---

## 硬约束

**此模块是纯只读的。在任何情况下，不得修改、删除、创建或移动任何文件。** 所有操作仅限于 Read、Glob、Grep 和只读 Bash 命令（`test -e`、`wc -l`、`cat`、`stat`）。

---

## 输入：4 层 CLAUDE.md 层级 + 辅助文件

扫描以下所有存在的文件，构建完整的上下文配置图谱：

### CLAUDE.md 四层层级

| 层级 | 标识 | 路径 |
|------|------|------|
| Managed | `managed` | `/etc/claude-code/CLAUDE.md` |
| User | `user` | `~/.claude/CLAUDE.md` |
| Project | `project` | `CLAUDE.md`、`.claude/CLAUDE.md`、`.claude/rules/*.md` |
| Local | `local` | `CLAUDE.local.md` |

### 辅助配置文件

| 类型 | 路径 |
|------|------|
| Memory 索引 | `MEMORY.md` |
| Memory 条目 | `memory/*.md`（位于 `~/.claude/projects/<project>/` 下） |
| 全局 settings | `~/.claude/settings.json` |
| 本地 settings | `.claude/settings.local.json` |

---

## 前置检查

1. 逐个检查上述所有路径是否存在（使用 `test -e`）。
2. 记录每个文件的存在状态，构建"已发现文件清单"。
3. 若所有文件均不存在 → 输出 `未发现任何上下文配置文件` 并结束。
4. 若至少有一个文件存在 → 按顺序执行以下 5 项检测。

---

## 检测 1：矛盾指令检测

读取所有已发现的 CLAUDE.md 层级文件和 memory 文件的完整内容。使用语义判断识别跨层级或跨文件的相互矛盾的规则。

**不确定性声明：** 矛盾检测和重复冗余检测（检测 3）依赖 Claude 的语义理解，属于不确定性行为——两次运行同一套文件可能得到不同报告。检测结果应视为启发性信号，不是权威判断。报告中每条矛盾都应标注"疑似"。

### 检测逻辑

1. Read 每个已发现文件的完整内容，记录每条规则/指令所在的文件路径和行号。
2. 对以下维度进行语义对比：
   - **语言指令矛盾** — 例如一层要求"始终用中文"，另一层要求"always respond in English"。
   - **编码风格矛盾** — 例如一处要求"不要加 docstring"，另一处要求"所有公开函数必须有 docstring"。
   - **行为指令矛盾** — 例如一处要求"每次修改后自动 commit"，另一处要求"不要自动 commit"。
   - **工具使用矛盾** — 例如一处要求"优先使用 Edit 工具"，另一处要求"始终用 Write 工具整体替换"。
   - **格式要求矛盾** — 例如一处要求"使用 tabs 缩进"，另一处要求"使用 2 空格缩进"。
3. 注意层级优先级：CC 的层级覆盖规则是 Local > Project > User > Managed。矛盾不一定是错误（低层可以有意覆盖高层），但仍应报告以供用户知晓。

### 输出

对每个发现的矛盾，报告：
- 两侧的源文件路径 + 行号 + 层级标识
- 矛盾内容描述

---

## 检测 2：死引用检测

扫描 settings.json、settings.local.json 和所有 CLAUDE.md/memory 文件中引用的路径，验证其是否存在。

### 检测逻辑

1. **settings.json 路径检测：**
   - 读取 `~/.claude/settings.json` 和 `.claude/settings.local.json`。
   - 提取 `permissions.allow` / `permissions.deny` 中引用的工具名或路径。
   - 提取 `hooks` 配置中引用的命令路径。
   - 提取任何 `path`、`command`、`file` 等字段中的文件系统路径。
   - 对每个提取到的路径，使用 `test -e` 验证是否存在。

2. **CLAUDE.md 和 memory 文件路径检测：**
   - 只检查结构化引用，不从自然语言正文中用正则提取路径（正文中的范例性路径会产生大量假阳性）
   - 检查范围限于：
     - `@path` / `@include` 引用指令
     - frontmatter 中的路径字段
     - 明确在代码块或行内代码中出现的绝对路径
   - 排除 URL（`http://`、`https://`）
   - 对每个提取到的路径，使用 `test -e` 验证是否存在

### 输出

对每个发现的死引用，报告：
- 源文件路径 + 行号
- 引用的不存在路径

---

## 检测 3：重复冗余规则检测

识别跨层级或跨文件的语义相同规则。

### 检测逻辑

1. 从所有已发现文件中提取可识别的规则/指令（通常表现为祈使句、带"必须"/"不要"/"始终"/"always"/"never"等关键词的句子）。
2. 对提取到的规则进行语义对比：
   - 完全相同的文本 → 明确重复
   - 不同措辞但含义相同 → 语义重复（例如"禁止使用 var"和"不要用 var，用 let/const"）
3. 仅报告跨不同文件的重复。同一文件内的重复不在本检测范围内。

### 输出

对每个发现的重复，报告：
- 两侧的源文件路径 + 行号 + 层级标识
- 表明两条规则语义等价

---

## 检测 4：体积异常检测

检查配置文件是否超出合理体积。

### 检测逻辑

使用 `wc -l < <file>` 获取行数，检查以下阈值：

| 检测对象 | 阈值 | 原因 |
|----------|------|------|
| 任意单个 memory 文件 | > 500 行 | 文件过大，可能导致上下文窗口浪费 |
| MEMORY.md 索引文件 | > 200 行 | CC 官方在 200 行后截断，超出部分不会被加载 |
| 任意单个 CLAUDE.md 层级文件 | > 500 行 | 过长的配置文件降低 Claude 遵循指令的准确度 |
| `.claude/rules/` 目录文件总数 | > 20 个 | 规则文件过多，增加矛盾和冲突风险 |

### 输出

对每个发现的体积异常，报告：
- 文件路径
- 实际行数
- 异常原因和建议

---

## 检测 5：远程托管设置冲突

CC 的 `remoteManagedSettings` 服务每小时轮询 `/api/claude_code/settings`，可以远程覆盖本地设置。本检测帮助用户区分"自己设置的"和"可能被远程覆盖的"配置。

### 检测逻辑

1. 读取 `~/.claude/settings.json`。
2. 检查是否存在 `remoteManagedSettings` 或类似的远程托管标记字段。
3. 已知的可被远程覆盖的字段包括：
   - `permissions.allow` / `permissions.deny`
   - `hooks`
   - `env`
   - `apiKeyHelper`
4. 对比本地 settings 与 settings.local.json：
   - 若某个字段在两者中都存在但值不同 → 可能存在覆盖冲突。
   - 若 `~/.claude/settings.json` 中有企业级字段（如 `remoteManagedSettings`）→ 该文件中的某些设置可能不是用户自己设置的。

### 输出

区分报告：
- **用户本地设置** — 用户在 `settings.local.json` 中明确设置的字段。
- **可能被远程托管的字段** — 在全局 `settings.json` 中存在且属于可被远程覆盖范畴的字段。
- **冲突字段** — 本地值与全局值不一致的字段。

---

## 输出格式

所有检测执行完毕后，输出完整诊断报告：

```
[scan-context 诊断报告]

矛盾指令：{N} 处
  - {file1}:{line} ({layer}) vs {file2}:{line} ({layer}) — {description}

死引用：{N} 处
  - {file}:{line} — 路径 {path} 不存在

重复冗余：{N} 处
  - {file1}:{line} ({layer}) ≈ {file2}:{line} ({layer})

体积异常：{N} 处
  - {file} 共 {N} 行，{reason}

远程托管：{status}
  - {details or "未检测到远程覆盖迹象"}

建议：{recommended actions}
```

若所有检测均健康，输出：「上下文健康，未发现问题。」

---

## 边界情况

- 所有配置文件均不存在 → 输出 `未发现任何上下文配置文件` 并结束。
- 某个检测的扫描对象全部不存在 → 该检测项报告 0 处发现，继续下一项。
- 语义判断的矛盾检测和重复检测存在假阳性风险 → 报告中标注"建议人工复核"。
- memory 目录路径需要动态定位 → 使用 `~/.claude/projects/` 下与当前项目对应的子目录。
- 远程托管设置检测无法直接访问远程 API → 仅基于本地文件中的痕迹推断，明确标注这是推断而非确认。
