# auto-shit — 输出风格配置

想换个说话方式？改这里就够了，工作流那边不用动。
只管搞笑，不用担心把马桶冲坏。

---

## 快速如厕输出模板

```
[auto-shit 如厕结果]

🧠 Memory：{N} 条
  - user_pref.md — [feedback] 用户偏好简洁回复，不要尾部总结 → 仍然相关 ✓
  - project_old.md — [project] 用户在开发 ShopNext，用 Prisma → 项目已不存在 ✗

💾 Sessions：{N} 个 transcript
  - abc123.jsonl (3.2 MB, 2天前) — 开发 auto-shit skill，调整 flush-memory 死引用检测逻辑 → 当前项目 🕐
  - def456.jsonl (196K, 15天前) — 讨论 React Router v6 嵌套路由配置 → 无关 ✗

📄 Plans：{N} 份
  - 2026-04-07-auto-shit-design.md — "auto-shit Design Document" → 当前活跃 ✓

🔧 Skills：0 个幽灵配置
📦 Plugin Cache：{N} 个缓存副本（{X} MB） ✗
📁 .claude/：{状态}
🌿 Worktrees：{状态}

---
📋 建议冲水清单：
  1. memory/project_old.md — 死引用
  2. def456.jsonl (196K) — 无关 session
  3. Plugin Cache 4 个副本（28 MB）

注：.DS_Store 等系统文件不纳入清理清单（无害且会自动重建）。

直接说"冲水"清理以上全部，或者告诉我要增减哪些项目。
```

---

## 标注规则

- ✓ — 仍然相关，留着吧
- ✗ — 这东西还没冲掉？加进清单
- 🕐 — 3 天内改过，暂不建议清理（说"加上 N"可手动加入）

---

## 清单为空时的回复
以下提示词任选其一

```
环境干净，不用冲水。
你今天刚刚开始工作吧？不要这么急着拉屎！
是不是一直在摸鱼，没有什么需要清理的残留物哦～
```

## 可自定义的触发提示（仅说明用，不影响路由逻辑）

| 触发词 | 行为描述 |
|--------|----------|
| `该拉屎了` / `shit` / `拉屎` / `auto-shit` | 先蹲坑看看情况 |
| `冲水` / `flush` | 冲！ |
| `便秘了` / `堵了` / `体检` / `scan` | 深度诊断 |
| `大扫除` / `全冲` | 从头到脚冲一遍，每步等你点头 |
| `定点拉屎` / `每天帮我拉一次屎` | 约好固定如厕时间，规律健康 |
| `别拉了` / `取消定时` | 忍住了，取消定时任务 |
| `冲 memory` / `清记忆` | 把记错的、记歪的都清掉 |
| `冲 .claude` / `清项目垃圾` | 清理项目目录里的陈年积垢 |
| `冲 plans` / `清旧文档` | 那些做完没人看的文档，冲掉 |
| `冲 skills` / `清废弃工具` | 找出装了但没用的工具清单 |
| `冲 worktrees` | 那些分支都删了，目录还在？清 |
| `查异物` / `扫残留` | 项目里混进来的奇怪文件，揪出来 |
